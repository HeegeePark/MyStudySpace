# Commit message rules

- `Add a` : a 추가
- `Fix b` : b 수정
  - 원본에서 에러사항 수정 및 보완
- `Delete c` : c 삭제
- `Update d` : d 업데이트
  - 에러 수정이 아닌 기능 추가 및 강화

---

### 더 전문적인(?) 버전

- `refactor` : 새로운 기능이 아닌 코드 재정비 시 (ex: 변수 이름 바꾸기)
- `fix` : 빌드 스크립트의 버그 수정
- `feat` : 빌드 스크립트 기능을 위한 것이 아니라 사용자의 편의를 위한 새로운 기능
- `chore` : 새로운 기능 추가
- `docs` : 문서 변경
- `test` : 누락된 테스트 추가, 생산적인 코드 생성 X
