# Branch Rules

1. master
   - Final version branch (version 0.0)
2. Develop
   - feature merge, commit (version 0.0 feat_x, feat_y)
3. Release
   - Only bug fix branch (version 0.0)
4. Feature_x
   - Activity branch x : Activity name (ex: feature_map)
5. Hotfix
   - 배포한 버전에 긴급 수정 필요 시, `master` 브랜치에서 분기
